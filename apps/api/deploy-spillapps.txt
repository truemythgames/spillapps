Concept

- **Spill** = platform: `spill-api`, `spill-db`, `spill-media`, `spill-cms`, spillapps.com DNS.
- **Products** = Tea apps with their own `app_id` (e.g. `bible-tea`), mobile/web under `apps/<product>/`, same API + bucket with prefixed keys.

Secrets and env (before production traffic)

- OPENAI_API_KEY (chat): `npx wrangler secret put OPENAI_API_KEY` â€” never commit.
- Local API (`wrangler dev`): `apps/api/.dev.vars` with `JWT_SECRET`, `GOOGLE_CLIENT_ID`, optional `OPENAI_API_KEY` (gitignored). See `.dev.vars.example`.

D1 migrations

- New environments: from repo root, `npm run migrate:all:local --workspace=@spillapps/db` (or run 0001â†’0005 files in order).
- Remote: run each new `packages/db/migrations/*.sql` once with `wrangler d1 execute spill-db --remote --file=...`

Before deploy (R2)
------------------
- If deploy fails with code 10042, enable R2 in the Cloudflare account (Dashboard â†’ R2), then redeploy.
- Bucket `spill-media` is declared in wrangler.toml (Wrangler can create it on deploy). If you had objects in another bucket, copy them into `spill-media` before relying on production URLs.
